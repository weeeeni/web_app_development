# 產品需求文件 (PRD)：個人記帳簿

## 1. 專案概述
- **背景與動機**：現代人生活收支繁雜，經常無法掌握金錢流向。建立一個簡單好用的個人記帳簿系統，讓使用者能快速記錄日常收支，並隨時掌握目前的財務狀況與餘額。
- **目標用戶**：需要記帳的個人用戶。
- **核心價值主張**：以最簡單、直覺的方式幫助使用者記錄財務，輕而易舉地追蹤每一筆收入與支出，掌握實際餘額，避免過度消費。

## 2. 功能需求
系統主要提供的核心功能如下：
1. **新增收入**
2. **新增支出**
3. **刪除收入與支出**
4. **顯示所有收入支出**
5. **顯示餘額統計**

**使用者故事 (User Stories)：**
- 作為 **個人用戶**，我希望 **能夠新增一筆收入記錄（包含金額、日期、備註等）**，以便 **記錄我的資金進帳情況**。
- 作為 **個人用戶**，我希望 **能夠新增一筆支出記錄（包含金額、日期、備註等）**，以便 **追蹤我日常的花費項目**。
- 作為 **個人用戶**，我希望 **能夠刪除錯誤輸入的收入或支出記錄**，以便 **修正我的帳目，確保資料正確無誤**。
- 作為 **個人用戶**，我希望 **能夠在一個列表中看到所有過去的收入與支出明細**，以便 **回顧歷史的現金流動狀況**。
- 作為 **個人用戶**，我希望 **系統能自動計算並顯示總餘額、總收入與總支出**，以便 **隨時掌握我目前的可用資金狀況**。

## 3. 非功能需求
- **技術限制**：
  - 伺服器端：Flask (Python)
  - 模板引擎：Jinja2
  - 資料庫：SQLite (適合輕量級個人應用)
- **效能與安全考量**：
  - 頁面載入速度需迅速，操作反應需即時。
  - 需要防範基本安全問題，如 SQL 注入（透過 ORM 或參數化查詢預防）與 XSS（依賴 Jinja2 本身的跳脫機制）。
  - 單機版或簡易個人使用，資料庫妥善存放在本地端。

## 4. MVP (最小可行性產品) 範圍

| 功能分級 | 功能描述 |
| :--- | :--- |
| **Must Have (必須有)** | 新增收入、新增支出、顯示所有輸入支出列表、顯示餘額統計 (總收入/總支出/淨額) |
| **Should Have (應該有)** | 刪除收入與支出記錄功能、在列表中醒目區分收入與支出 (如顏色) |
| **Nice to Have (可以有)** | 編輯(修改)已儲存的記錄、依據月份或類別篩選資料、收支圓餅圖表分析 |
